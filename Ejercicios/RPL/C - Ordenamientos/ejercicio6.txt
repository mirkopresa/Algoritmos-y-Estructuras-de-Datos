Realizar el seguimiento de ordenar por Radix Sort el siguiente arreglo de cadenas que representan versiones. 
Cada versión tiene el formato a.b.c, donde cada valor a, b y c puede tener un valor entre 0 y 99. 
Considerar que se quiere que quede ordenado primero por la primera componente (a), 
luego por la segunda (b) y finalmente por la tercera (c). 
Se puede asumir que a nunca será 0 salvo que el número sea efectivamente 0. 
Es decir, la notación es correcta. Tener en cuenta que, por ejemplo 1.1.3 < 1.1.20, 2.20.8 < 3.0.0.

Arreglo: ["4.3.2", "5.1.2", "10.1.4", "2.1.20", "2.2.1", "4.2.3", "2.1.5", "8.1.2", "5.30.1", "10.0.23"]

Vamos a ordenar con Radix Sort 3 veces, utilizando Counting Sort como ordenamiento auxiliar para que sea estable
En cada ejecucion de Counting Sort, estaremos enviando una parte diferente de la cadenas
Empezaremos ordenando del menos significativo al mas significativo, es decir c -> b -> a
Counting Sort recibira el arreglo, el rango que en este caso es 100, y una funcion que extraiga del string
lo que necesitemos
Primero enviamos el arreglo, rango 100 y una funcion que nos devuelve el caracter "c" de manera numerica
Como estamos haciendo un counting sort, primero contamos las frecuencias, luego calculamos las sumas acumuladas, y por ultimo
ordenamos con estos datos
El arreglo de frecuencias nos quedaria [0, 2, 3, 1, 1, ..., 1 (20), ..., 1 (23), ...]
Luego las sumas acumuladas [0, 2, 5, 6, 7, ..., 8, ..., 9, ....]
Y ordenamos chequeando las sumas acumuladas, para saber cual es la posicion inicial de cada numero
Repetimos este proceso 2 veces mas, enviandole b, y por ultimo a

El arreglo ordenado seria: ["2.1.5", "2.1.20", "2.2.1", "4.2.3", "4.3.2", "5.1.2", "5.30.1", "8.1.2", "10.0.23", "10.1.4"]
